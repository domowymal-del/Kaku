02 · 助手

失败的命令，不该只是一行红字。

命令失败时，Kaku 给出可审查的修复建议，按 Cmd + Shift + E 放回命令行，危险命令绝不自动执行。输入 # 加一句话，还能把自然语言变成命令。

了解助手

~/proj $ npm run buidl

npm ERR! Missing script: "buidl"

Kaku Assistant · 建议修复
npm run build
按 ⌘⇧E 放回命令行，不会自动执行。

~/proj $ npm run build

04 · 日常入口

真正会每天用到的入口。

只保留每天会碰到的入口。

01

工具

Cmd + Shift + G 打开 Lazygit，Cmd + Shift + Y 打开 Yazi，SSH 会话里用 Cmd + Shift + R 挂载远端文件。

02

聊天

Cmd + L 打开内建 AI Chat；shell 里用 k 或 kaku chat 继续同一套对话和记忆。

03

诊断

kaku doctor 检查 app bundle、PATH、shell 集成和可选工具；kaku init 负责 zsh/fish 集成和常用工具。

05 · 为什么是 Kaku

一个终端，应该先让你进入工作。

我重度依赖终端。用了多年 Alacritty 图它快，后来工作转向 AI 编码，想要更顺手的标签和分屏。Kaku 保留 WezTerm 的引擎，加上实用的默认值和 Shell 工具，让助手触手可及，全都装进一个快速、安静的 macOS 应用。

06 · 问答

常见问题

怎么安装？
从 GitHub Releases 下载 DMG，或用 Homebrew tap：brew install tw93/tap/kakuku，Homebrew 上曾有无关的 kaku 包，使用 tap 可以避免装错。
Kaku 和 iTerm2 有什么区别？
iTerm2 是成熟的 macOS 终端。Kaku 是更轻的 WezTerm 分支，带实用默认值、顺手的标签和分屏、一套精选 Shell 工具，并内建 AI 助手，开箱即用。
Kaku 和 Warp 有什么区别？
Warp 是闭源并偏云端账号体系。Kaku 是 MIT 开源，不需要账号，AI 请求只发往你配置的 Provider。
Kaku 收费吗？
不收费，Kaku 是 MIT 协议的开源项目。
支持 Windows 或 Linux 吗？
暂时不支持，当前重点是先把 macOS 原生体验打磨稳定。
