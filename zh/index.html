02 · 助手

失败的命令，不该只是一行红字。

命令失败时，Kaku 给出可审查的修复建议，按 Cmd + Shift + E 放回命令行，危险命令绝不自动执行。输入 # 加一句话，还能把自然语言变成命令。

了解助手

~/proj $ npm run buidl

npm ERR! Missing script: "buidl"

Kaku Assistant · 建议修复
npm run build
按 ⌘⇧E 放回命令行，不会自动执行。

~/proj $ npm run build

04 · 日常入口

先从真正有用的路径开始。

安装、配置 AI，再按自己的习惯微调终端，不需要先读完整文档。

01

安装

用 DMG，或运行 brew install tw93/tap/kakuku，再用 kaku doctor 检查 app、PATH 和 shell 集成。

02

AI 设置

运行 kaku ai，选择 Codex、OpenAI 或自定义 Provider。配置好后，Cmd + L 打开 AI Chat。

03

工具

Cmd + Shift + G 打开 Lazygit，Cmd + Shift + Y 打开 Yazi，SSH 会话里用 Cmd + Shift + R 挂载远端文件。

04

配置

打开 kaku config 调整字号、透明度、Smart Tab、关闭确认、快捷键和 Lua 配置。

05 · 为什么是 Kaku

一个终端，应该先让你进入工作。

我重度依赖终端。用了多年 Alacritty 图它快，后来转向 AI 编码，想要更顺手的标签和分屏。Kaku 基于一套快速可靠的引擎，加上实用默认值和 Shell 工具，让助手触手可及，全都装进一个快速安静的 macOS 应用。

06 · 问答

常见问题

怎么安装？
从 GitHub Releases 下载 DMG，或用 Homebrew tap：brew install tw93/tap/kakuku，Homebrew 上曾有无关的 kaku 包，使用 tap 可以避免装错。
Kaku 和 iTerm2 有什么区别？
iTerm2 是成熟的 macOS 终端。Kaku 更轻更快，带实用默认值、顺手的标签和分屏、一套精选 Shell 工具，并内建 AI 助手，开箱即用。
Kaku 和 Warp 有什么区别？
Warp 是闭源并偏云端账号体系。Kaku 是 MIT 开源，不需要账号，AI 请求只发往你配置的 Provider。
Kaku 收费吗？
不收费，Kaku 是 MIT 协议的开源项目。
支持 Windows 或 Linux 吗？
暂时不支持，当前重点是先把 macOS 原生体验打磨稳定。
