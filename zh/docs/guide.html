第一次用 Kaku？这一页按平时上手的顺序过一遍，从第一个窗口讲到 AI 助手。要查具体细节，看功能与 AI、快捷键和配置。

#第一次打开

打开 Kaku，就是一个终端窗口加一个 shell 提示符，直接能输入。没有设置向导，也不用注册账号。

  • 主题跟随 macOS。Kaku 按系统外观启动深色或浅色，系统切换时它也跟着切。
  • 窗口按钮在标签栏里。macOS 的关闭、最小化、缩放按钮收进了标签栏左上角，窗口更干净。
  • 标签栏会自动隐藏。只有一个标签时屏幕上没有多余的东西，开第二个标签时它才出现。

如果之后 shell 找不到 kaku 命令，运行 kaku doctor 检查环境。首次运行细节见安装。

#标签与分屏

标签和分屏用的都是你熟悉的 macOS 快捷键，基本不用打开菜单。

想做什么快捷键结果
开新标签Cmd + T在同一目录新增一个标签，并显示标签栏。
左右分屏Cmd + D把当前面板分成并排的两个。
上下分屏Cmd + Shift + D在当前面板下方叠一个新面板。
在面板间切换Cmd + Opt + 方向键焦点移到那个方向的面板。
关闭面板 / 标签Cmd + W关掉当前面板；若是最后一个面板则关标签，否则隐藏应用。
重开刚关的标签Cmd + Shift + T恢复你最后关掉的标签，连同原来的目录。
重命名标签双击标签标题输入名字，回车保存。

想把同一段内容同时敲进多个面板？按 Cmd + Opt + I 把按键广播到当前标签的每个面板。完整列表见快捷键。

#开箱即用的 Shell

Kaku 首次启动就配好了一套 zsh（也支持 fish），不用你动配置文件。

  • 自动建议。边输入时，光标前会冒出一条来自历史记录的灰色建议。
  • Smart Tab。有灰色建议时按 Tab 接受它，没有时按 Tab 打开补全列表。这是默认行为，可在配置里切换。
  • 快速跳目录。输入 z proj 直接跳到你常去的目录，不用敲完整路径。
  • 彩色输出。命令和报错会随输入着色；装了 Delta 时 git diff 用它分页。

还有两个不用按键的小动作：选中文字一松开鼠标就复制到剪贴板；Cmd + 点击用默认应用打开文件路径或网址。

#AI 助手

助手是可选的，只有你主动用时才会运行。先用 kaku ai 配置一次：打开开关，选一个 provider（复用你的 codex 登录、填一个 OpenAI key，或指向自定义端点），就好了。之后有三种用法。

  1. 修复失败的命令。命令报错退出时，Kaku 会在提示符下方给出一条修复建议。按 Cmd + Shift + E 把它粘进来。像 rm -rf 这类危险命令会粘出来给你看，但绝不会自己执行。
  2. 一句话变命令。输入 # 加一句话，比如 # 找到并杀掉占用 3000 端口的进程，回车。Kaku 把它变成真正的命令放到提示符上，等你确认再运行。
  3. 打开聊天面板。按 Cmd + L 打开聊天，回答会流式输出、代码高亮，还能读取当前终端上下文。在任意 shell 里 k "..." 或 kaku chat 打开的是同一段对话。

请求只会发给你配置的那个 provider。完整设置细节见功能与 AI。

#内置工具

几个常用的终端工具，一个快捷键就能调出来，都在当前面板里打开，退出后回到 shell。

工具快捷键作用
LazygitCmd + Shift + G当前仓库的可视化 git 界面。
YaziCmd + Shift + Y文件管理器，退出时把你带到选中的目录。
远程文件Cmd + Shift + R把当前 SSH 主机的文件挂到本地并浏览。

缺哪个工具，运行 kaku init，Kaku 会问你要不要装。主题和行为细节见功能与 AI。

#设置与体检

Kaku 自带合理的默认值，你想改多少改多少。

  • 快速设置。按 Cmd + , 打开设置面板，调滚动条、Smart Tab 模式、AI 模型等常用项。
  • 完整配置。运行 kaku config 打开 ~/.config/kaku/kaku.lua，可以设字号、窗口透明度、自定义快捷键和任意 Lua 覆盖。见配置。
  • 检查环境。感觉哪里不对就运行 kaku doctor，它会检查应用包、PATH、shell 集成和可选工具。

日常用这些就够了。想深入设置或查具体快捷键，翻其余几页文档。
