Error recovery

失败的命令，不再只是一行红字。

Kaku 会把失败命令、退出码、当前目录和 git 分支交给 Assistant 生成修复建议。按 Cmd + Shift + E 才会把建议送回命令行，危险命令永远需要你审查。

~/proj $ npm run buidl

~/proj $ npm run build ok

默认值已经替你做过选择。

不是另一个需要你花一整晚配置的终端。Kaku 把字体、配色、Shell 工具、AI 入口和 WezTerm 兼容性整理成一套可直接工作的默认体验。

Natural language to command

让终端理解一句人话，但不要替你偷偷执行。

在提示符前输入 # 加自然语言，Kaku 调用你配置的 LLM，把生成命令注入回提示符。它负责起草，你负责确认。

工具

诊断

kaku doctor 检查 app bundle、PATH、shell 集成和可选工具；kaku init 负责 zsh/fish 集成和常用工具。

FAQ

常见问题

Kaku 和 iTerm2 有什么区别？

iTerm2 是成熟的 macOS 终端。Kaku 更聚焦 AI 编码工作流，内建 AI Chat、错误修复、自然语言转命令，并保留 WezTerm 的可定制底座。

Kaku 和 Warp 有什么区别？

Warp 是闭源并偏云端账号体系。Kaku 是 MIT 开源，不需要账号，AI 请求只发往你配置的 Provider。

Kaku 收费吗？

不收费。Kaku 是 MIT 协议的开源项目。

支持 Windows 或 Linux 吗？

暂时不支持。当前重点是先把 macOS 原生体验打磨稳定。

Homebrew 装哪个包？

使用 brew install tw93/tap/kakuku。Homebrew 上曾有无关的 kaku 包，使用 tap 可以避免装错。
