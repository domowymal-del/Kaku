Error recovery

失败的命令，不再只是一行红字。

Kaku 会在命令失败后给出可审查的修复建议。你仍然保留控制权，按下快捷键才会把建议应用回命令行。

Natural language to command

让终端理解一句人话，但不要替你偷偷执行。

在提示符前输入 # 加自然语言，Kaku 调用你配置的 LLM，把生成命令注入回提示符。它负责起草，你负责确认。

Migration

从 iTerm2 或 WezTerm 迁移，不应该像搬家。

Kaku 保留常见终端习惯，也尊重 WezTerm 用户已有的 Lua 配置。迁移重点不是重学，而是把 AI 编码路径补上。

  • 保留熟悉的 Cmd+T、Cmd+W、Cmd+D 操作习惯
  • 获得更轻的启动体验和内建 AI 助手
  • 继续使用 macOS 原生分发和公证应用
完整迁移指南
  • 复用 WezTerm Lua 配置和渲染底座
  • 更小体积，更贴近 AI 编码默认工作流
  • 保留可定制性，不牺牲日常易用性

FAQ

常见问题

Kaku 和 iTerm2 有什么区别？

iTerm2 是成熟的 macOS 终端。Kaku 更聚焦 AI 编码工作流，内建错误修复、自然语言转命令，并保留 WezTerm 的可定制底座。

Kaku 和 Warp 有什么区别？

Warp 是闭源并偏云端账号体系。Kaku 是 MIT 开源，不需要账号，AI 请求只发往你配置的 Provider。

Kaku 收费吗？

不收费。Kaku 是 MIT 协议的开源项目。

支持 Windows 或 Linux 吗？

暂时不支持。当前重点是先把 macOS 原生体验打磨稳定。
